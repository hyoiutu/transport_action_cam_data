# Branch Rules (ブランチ運用規約)

本ファイルは、AIエージェントがGitHub Issueをもとに実装を進める`issue-implement`スキルを実行する際に遵守すべき、ブランチ運用の詳細なルールを定義します。

---

## 🚫 絶対遵守ルール (Mandatory Rules)

0. **1 Issue = 1 Branch**
   - 1つのGitHub Issueに対して、必ず1つの専用ブランチを割り当てること。
   - 複数のIssueを同一ブランチで同時に扱ってはならない。
   - 対象Issue用のブランチが既に存在する場合は新規作成せず、そのブランチをチェックアウトして再利用すること。

1. **プッシュの絶対禁止**
   - 本スキルまたはAIエージェントの処理を通じて、リモートリポジトリへのプッシュ (`git push`) を絶対に実行してはならない。

2. **ブランチ作成前のユーザー確認**
   - ブランチを作成する前に、`issue-implement`スキルのプロセスに従い、以下をユーザーに確認し合意を得ること。
     - 対象とするGitHub Issue番号
     - ブランチ名
     - 派生元ブランチ

3. **ブランチ名の決定**
   - ユーザーがブランチ名を明示的に指定した場合は、その名前をそのまま使用する。
   - 指定がない場合、AIは以下の形式でブランチ名を自動生成する。
     ```text
     <type>/issue-<Issue番号>-<内容を表す英語スラッグ>
     ```
     - `<type>`: [commit_rules.md](./commit_rules.md)のコミットメッセージプレフィックス規約（`feat`/`fix`/`docs`/`style`/`refactor`/`test`/`chore`）に準じて、Issueの内容から最も適切なものを選択する。
     - `<内容を表す英語スラッグ>`: Issueタイトルを要約した、英語の小文字・ハイフン区切り文字列（kebab-case）。
     - 例:
       - `feat/issue-10-github-issue-skill`
       - `fix/issue-9-disable-ui-during-copy`
       - `fix/issue-8-video-creation-time`

4. **派生元ブランチ**
   - デフォルトでは、スキル起動時にAIエージェントが作業しているカレントブランチから分岐する。
   - ユーザーが別のブランチ（例: `main`）を指定した場合は、そちらから分岐する。
   - 派生元ブランチに未コミットの変更が残っている場合は、ブランチを切る前にユーザーに状況を伝え、確認を得ること（コミットする場合はauto-commitスキル経由とする）。

5. **ブランチ作成コマンド**
   - 新規作成時: `git checkout -b <ブランチ名> <派生元ブランチ>`
   - 既存ブランチの再利用時: `git checkout <ブランチ名>`

6. **実装後のコミット**
   - ブランチ上で実装した変更をコミットする際は、必ず[auto-commitスキル](./commit_rules.md)の手順（本ファイルではなく`commit_rules.md`に従う）を経由すること。本スキルおよびAIエージェントが直接`git commit`を実行してはならない。
