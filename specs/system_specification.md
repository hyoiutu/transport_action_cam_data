# 仕様書: Action Cam Data Transporter

本仕様書は、アクションカメラ（Sony, DJI等）で撮影された動画および画像データを、撮影日時に基づいて自動的に整理し、外付けHDDなどの指定フォルダにコピーするElectron製デスクトップアプリケーションの要件および設計仕様を定義します。

---

## 1. システム概要
- **製品名**: Action Cam Data Transporter (仮)
- **プラットフォーム**: デスクトップアプリケーション (Electron)
- **サポートOS**: macOS, Windows (OS間の差異を吸収し、同様の操作性を提供)
- **目的**: 撮影された大容量の動画・画像データを、撮影日に基づいて `YYYY-MM-DD/` のフォルダに自動で分類・整理してバックアップする。

---

## 2. 主要機能要件

### 2.1. フォルダ選択・パス管理
- **コピー元（Source）フォルダ選択**:
  - ローカルディレクトリまたはマウントされたSDカード/カメラデバイスのフォルダを選択可能。
  - ドラッグ＆ドロップまたはOS標準 of フォルダ選択ダイアログで指定。
- **コピー先（Destination）フォルダ選択**:
  - 外付けHDD等のバックアップ先フォルダを選択可能。
  - 同様にダイアログまたはドラッグ＆ドロップで指定。
- **パスの正規化**:
  - macOS/Windows のパス区切り文字の差異をNode.jsの `path` モジュールで自動吸収。
- **フォルダ表示とナビゲーション**:
  - コピー元・コピー先フォルダ直下のサブディレクトリは、対応フォーマットのファイルと並べてフォルダとして一覧表示される（フォルダを先に、ファイルをあとに、それぞれ名前順で表示）。これにより、コピー実行後に自動生成される撮影日ごとのサブフォルダ（`YYYY-MM-DD/`）の中身をアプリ内から確認できる。
  - フォルダをクリックすると、そのフォルダへ移動し、中身（ファイル・フォルダ）を一覧表示する。移動すると、コピー元/コピー先フォルダそのもの（サイドバーのパス表示）が移動先のフォルダに更新される。そのため、移動後に「コピーを開始する」を押すと、現在表示しているフォルダを基準にコピーが行われる（コピー自体は従来通り非再帰的で、直下のファイルのみが対象）。
  - サイドバーの「一つ上へ」操作で、1階層上の親フォルダに戻ることができる（移動範囲の制限はなく、元々選択したフォルダより上の階層にも移動できる）。

### 2.2. ファイル自動分類・コピー機能
- **対応フォーマット**:
  - 動画: `.mp4`, `.mov`, `.avi`, `.mkv`（拡張子の大文字小文字は区別しない）
  - 画像: `.jpg`, `.jpeg`, `.png`（拡張子の大文字小文字は区別しない）
  - 上記以外の拡張子のファイルはスキャン対象外とする（エラーや警告は表示せず、単に一覧・コピー対象から除外される）。フォルダ内のサブディレクトリはコピー対象・件数のカウントには含まれないが、フォルダとして一覧に表示・移動できる（詳細は2.1を参照）。詳細は「2.4. エラー・異常系の挙動」も参照。
- **撮影日時の取得（OS非依存）**:
  - **動画**: 動画ファイルのメタデータ（`creation_time`）をNode.jsライブラリ（`music-metadata` 等）を用いて抽出。
  - **画像**: EXIF情報（`DateTimeOriginal`）をNode.jsライブラリ（`exif-parser` 等）を用いて抽出。
  - **フォールバック（メタデータ欠落時）**: メタデータから撮影日時が取得できない場合、ファイルの更新日時（`mtime`）または作成日時（`birthtime`）を撮影日時とみなして処理する。
- **フォルダ自動生成**:
  - コピー先フォルダ配下に `YYYY-MM-DD/` 形式のフォルダを自動作成。
- **重複ファイル処理（競合回避）**:
  - コピー先に同名ファイルが既に存在する場合、ファイル名の末尾に連番を付与して保存する（例: `MAH0001.MP4` が既に存在する場合は `MAH0001_1.MP4`）。
- **コピーの挙動**:
  - コピーのみを行う（コピー元のファイルはそのまま残す）。
- **進捗表示**:
  - 大容量ファイルのコピー進捗（全体およびファイルごと）をパーセンテージと進捗バーでUIにリアルタイム表示。
  - コピー処理のキャンセル機能。

### 2.3. 動画・画像プレビュー機能
- **ギャラリービュー**:
  - コピー元およびコピー先に存在する動画・画像をサムネイル一覧で表示。
- **インラインプレビュー**:
  - 画像をクリックした際に、モーダル等で拡大表示。
  - 動画をクリックした際に、HTML5 `<video>` 要素を用いてアプリ内で直接再生。

### 2.4. エラー・異常系の挙動

- **スキャン結果が0件の場合**:
  - フォルダが未選択、存在しない、または対応フォーマット（2.2参照）のファイルが1件も無い場合は、エラーとしては扱わず「0件」の状態として扱う。
  - ギャラリービューには空状態メッセージ（コピー元: 「表示するファイルがありません。コピー元フォルダを選択してスキャンしてください。」／コピー先: 「コピー先フォルダに動画や画像がありません。」）を表示する。
  - コピー元が0件、またはコピー先フォルダが未選択の間は、コピー開始ボタンを無効化する。

- **コピー処理のキャンセル機能（詳細仕様）**:
  - キャンセル判定はファイル単位で行われる。ユーザーが「キャンセル」を押した時点で処理中の1ファイルはコピーが完了するまで中断されない（コピーの途中で強制停止されるわけではない）。判定は次のファイルの処理を開始する直前に行われ、以降のファイルはコピーされない。
  - 既にコピーが完了したファイルは削除・ロールバックされず、コピー先にそのまま残る。
  - キャンセル機能に「再開（レジューム）」の概念はない。キャンセル後に同じ操作をやり直した場合、対象ファイル一覧全体に対して改めてコピーが実行される。その結果、キャンセル前に既にコピー済みだったファイルは、重複ファイル処理（2.2参照）により連番（`_1`等）を付与された別ファイルとして再度コピーされる。

- **その他のエラー（読み取り権限がない、ディスク容量不足、コピー先デバイスの切断等）**:
  - フォルダのスキャンに失敗した場合（読み取り権限がない等）: エラーダイアログを表示し、対象の一覧は0件になる。
  - コピー処理の開始前に失敗した場合（コピー先フォルダが存在しない・削除された等）: コピー処理全体を中断し、進捗表示に「エラーが発生しました」と表示するとともに、エラーダイアログを表示する。
  - コピー処理の実行中に個別のファイルで失敗した場合（読み取り権限がない、ディスク容量不足、対象ファイルの消失、コピー先デバイスの切断等）: そのファイルのみをスキップ（コピー成功件数にはカウントしない）し、エラーダイアログを表示した上で、残りのファイルの処理を継続する。そのため、コピー先デバイスが処理の途中で丸ごと利用できなくなった場合は、残りの全ファイル分のエラーダイアログが1件ずつ連続して表示され、最終的にはコピー成功件数が少ない状態のまま「コピー完了」として処理を終える（処理全体が「失敗」として終了するわけではない）。

- **コピー処理中の操作制限**:
  - コピー処理中は、コピー元・コピー先フォルダの再選択（クリックまたはドラッグ＆ドロップ）と、コピー開始ボタンを無効化し、多重実行を防止する。

- **アプリ終了時の挙動（既知の制限）**:
  - コピー処理中にウィンドウを閉じる・アプリを終了することを防ぐ仕組みは現状実装されていない。コピー処理中にアプリを終了した場合、書き込み中だったファイルが不完全な状態のままコピー先に残る可能性がある。

## 3. UI/UX デザインコンセプト
- **モダン・プレミアムなUI**:
  - ダークモードを基調とした、直感的で洗練されたインターフェース。
  - ガラスモルフィズム（透過エフェクト）やスムーズなアニメーションの適用。
- **直感的なダッシュボード**:
  - 左ペイン：コピー元・コピー先のフォルダ選択、ステータス表示。
  - 中央/右ペイン：ファイルプレビュー（サムネイルグリッド）。
  - 下部：コピー実行ボタンおよび進行状況表示（プログレスバー）。

---

## 4. 技術スタック
- **フレームワーク**: Electron
- **フロントエンド**: React, HTML5, CSS3 (既存CSSによるカスタムスタイリング), TypeScript (ESM)
- **ビルド・コンパイルツール**: Vite (レンダラービルド), tsc (メインプロセス・プリロードスクリプトのコンパイル)
- **メタデータ解析**:
  - 動画: `music-metadata` (純粋なJavaScript/Node.js実装で、外部バイナリ `ffprobe` のインストール不要)
  - 画像: `exif-parser` 等のNode.jsライブラリ
- **アイコン**: `lucide-react` によるモダンなSVGアイコン
- **Lint / Format**: Biome（`rules.md` のコーディング規約に基づく）
- **git commit時の自動チェック**: husky + lint-staged による lint・型チェックの自動実行

## 5. 検証方針
- **単体テスト**: Vitest + Testing Library を使用し、対象ファイルは `src/**/*.tests.*` および `electron/**/*.tests.*` に限定する。テスト対象ファイルと同じディレクトリ内の `__tests__/` に配置する。ルールの詳細は `test_rules.md` を参照。
- **E2Eテスト**: Playwright を使用し、`src/tests/` 配下に配置する。
