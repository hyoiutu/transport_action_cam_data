# 仕様書: AI自動コミットスキル

本仕様書は、Google Antigravity環境において、AIエージェントがローカルファイルに変更を加えた際、自動的にコミット処理を行うカスタムスキルの要件および設計仕様を定義します。

---

## 1. システム概要
- **名称**: auto-commit
- **目的**: AIエージェントが作業中にローカルファイルに変更を加えた場合、コミットルールに準拠した形で安全かつ自動的にGitコミットを実行する。
- **動作対象**: Google Antigravity CLI / IDE / 2.0 にて稼働するAIエージェント。

---

## 2. 主要機能要件

### 2.1. 自動起動と変更検知
- AIエージェントは、何らかのタスクによりローカルファイルに変更（ステージングされていない変更、またはステージング済みの変更）が発生したことを検知した際、自動的に本スキルの処理プロセスを起動しなければならない。

### 2.2. ルールファイルの参照 (`commit_rules.md`)
- コミット処理を実行する前に、プロジェクトのルートディレクトリに配置された `commit_rules.md` を読み込み、そこに記載されているルールを厳格に遵守しなければならない。

### 2.3. テスト・リント・型チェックの実行と検証
- コミットを行う前に、`commit_rules.md`の「開発プロジェクト固有の検証手順」に記載された全てのコマンド（テスト・リント・型チェック等）を必ず実行する。
- **例外なしルール**: いずれかが通らない場合はコミットを行ってはならず、エラーを解決するかコミットプロセスを中断しなければならない。

### 2.4. コミットメッセージの生成
- 変更内容（`git diff`）を分析し、要約した1行程度（簡潔で明確）のコミットメッセージを自動生成する。

### 2.5. ユーザーへの確認と決定権の委譲
- 実際に Git コミットを実行する前に、以下の情報をユーザーにチャットで提示し、最終的なコミットの可否について「Y/N」の確認を求めなければならない。
  - **コミット対象のファイル一覧**
  - **生成したコミットメッセージ**
  - **コミット対象のブランチ名**
- ユーザーから「Y」または肯定的な同意を得た場合のみコミットを実行する。否定的な回答、またはメッセージ修正の指示があればそれに従う。

### 2.6. プッシュの禁止
- **絶対ルール**: 本スキルおよびAIエージェントのプロセス経由で `git push` を実行することは絶対に禁止する。

---

## 3. ファイル構造と配置場所

プロジェクト内の以下の構造でカスタムスキルおよび関連ファイルを作成します。

```text
./ (プロジェクトルート)
├── .agents/
│   └── skills/
│       └── auto-commit/
│           └── SKILL.md            # スキルのインストラクションファイル（実体）
├── .claude/
│   └── skills/
│       └── auto-commit/
│           └── SKILL.md            # .agents/skills/auto-commit/SKILL.md へのシンボリックリンク
├── commit_rules.md                 # ユーザーが編集可能なコミットの細則ルールファイル
└── specs/
    └── auto_commit_skill_specification.md  # 本仕様書
```

### 3.1. Claude Code との互換性
本スキルは元々 Google Antigravity 環境向けに設計されたが、SKILL.md の YAML フロントマター形式（`name` / `description`）が Claude Code のスキル仕様とも互換性があるため、`.claude/skills/auto-commit/SKILL.md` にシンボリックリンクを配置することで Claude Code からも `/auto-commit` として呼び出せるようにする。実体は `.agents/skills/auto-commit/SKILL.md` の一箇所のみとし、二重管理を避ける。

---

## 4. 各ファイルの定義

### 4.1. `commit_rules.md` (プロジェクトルート)
ユーザーがコミット時の細則を定義するマークダウンファイル。デフォルトで以下のルールを記載する：
1. このスキル（およびAI）経由での `git push` は絶対に実行しないこと。
2. テスト・リント・型チェックが通らない場合はそれを絶対に無視せず、エラーを解消するかコミットを中断すること。
3. コミットメッセージは1行程度で要約し、簡潔かつ分かりやすいものにすること。

### 4.2. `.agents/skills/auto-commit/SKILL.md`
AIエージェントが本スキルをどのように解釈し、実行するかを規定した指示書。YAMLフロントマターを含める。
